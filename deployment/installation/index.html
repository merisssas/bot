        Installation and Updates #

        Deploy from Pre-compiled Binary (Recommended) #

        Download the binary file for your platform from the Release page.

        Create a config.toml file in the extracted directory, refer to the Configuration Guide to edit the configuration file.

        Run:

        chmod +x Teleload
./Teleload

        Daemon #

        Create a file /etc/systemd/system/Teleload.service and write the following content:
        [Unit]
Description=Teleload
After=systemd-user-sessions.service

[Service]
Type=simple
WorkingDirectory=/yourpath/
ExecStart=/yourpath/Teleload
Restart=always

[Install]
WantedBy=multi-user.target
        Enable startup on boot and start the service:
        systemctl enable --now Teleload

        Add Boot Autostart Service

        Create a file /etc/init.d/Teleloadbot, refer to wrt_init and modify as needed:
        #!/bin/sh /etc/rc.common

#This is the OpenWRT init.d script for Teleload

START=99 
STOP=10
description=&#34;Teleload&#34;

WORKING_DIR=&#34;/mnt/mmc1-1/Teleload&#34;
EXEC_PATH=&#34;$WORKING_DIR/Teleload&#34;
start() {
    echo &#34;Starting Teleload...&#34;
    cd $WORKING_DIR
    $EXEC_PATH &amp;
}
stop() {
    echo &#34;Stopping Teleload...&#34;
    killall Teleload
}
reload() {
    stop
    start
}
        Set permissions:
        chmod +x /etc/init.d/Teleloadbot
        Then copy the file to /etc/rc.d and rename it to S99Teleloadbot, also set permissions:
        chmod +x /etc/rc.d/S99Teleloadbot

        Add Shortcut Commands

        Create a file /usr/bin/sabot, refer to wrt_bin and modify as needed. Note that the file encoding here only supports ANSI 936. Then set permissions:
        chmod +x /usr/bin/sabot
        Usage: sudo sabot start|stop|restart|status|enable|disable

        Deploy Using Docker #

        Docker Compose #

        Download the docker-compose.yml file, create a new config.toml file in the same directory, refer to config.example.toml to edit the configuration file.

        Start:

        docker compose up -d

        Docker #

        docker run -d --name Teleload \
    -v /path/to/config.toml:/app/config.toml \
    -v /path/to/downloads:/app/downloads \
    ghcr.io/merisssas/bot:latest
        About Docker image variants
        • Default: Includes all features and dependencies, larger in size. Use this if you don't have special requirements.
        • micro: Slimmed-down image with some optional dependencies removed, smaller in size.
        • pico: Minimal image containing only core features, smallest in size.
        You can pull different variants by specifying tags, for example: ghcr.io/merisssas/bot:micro
        For more details about the variants, see the Dockerfile in the project root.

        Updates #

        If you deployed from pre-compiled binaries, use the following CLI command to update:

        ./Teleload up

        (upgrade is also available as an alias.)

        If you deployed with Docker, use the following commands to update:

        docker:

        docker pull ghcr.io/merisssas/bot:latest
docker restart Teleload

        docker compose:

        docker compose pull
docker compose restart
